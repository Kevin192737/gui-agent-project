# 算法设计说明文档

## 一、方案概述

本方案采用“视觉优先 + 任务阶段约束 + 动作后处理”的策略来完成移动端 GUI 任务。整体流程为：

1. 读取当前截图和历史动作。
2. 根据用户指令构建任务画像，识别应用名、搜索词、评论内容等关键信息。
3. 结合历史动作推断当前阶段，限制可用动作类型。
4. 调用多模态模型输出下一步动作。
5. 对模型输出进行格式校验与后处理，保证动作类型、参数结构和坐标范围符合评测要求。

## 二、核心模块

### 1. 任务画像构建

通过规则与正则抽取以下信息：
- 目标应用名
- 搜索词
- 评论内容
- 起点/终点城市
- 是否需要收藏、点赞页、评论区等

### 2. 阶段推断

根据历史动作和任务类型，判断当前处于：
- 打开应用
- 搜索输入
- 结果确认
- 内容打开
- 评论输入/提交
- 地图路线规划

### 3. 动作约束与后处理

对模型输出进行如下约束：
- CLICK 仅允许输出归一化中心点 `point`
- TYPE 仅允许输出 `text`
- OPEN 仅允许输出 `app_name`
- SCROLL 仅允许输出起止点
- COMPLETE 不携带参数

同时对搜索、评论、地图等常见任务加入规则化修正，避免常见的动作类型漂移和输入重复。

## 三、优化思路

- 提升搜索类任务的输入阶段稳定性，减少本应 `TYPE` 却输出 `CLICK` 的情况。
- 对高频应用（如爱奇艺、京东、拼多多等）加入更明确的阶段约束。
- 对地图、评论、收藏等任务加入更强的状态机式约束，减少无效点击。
